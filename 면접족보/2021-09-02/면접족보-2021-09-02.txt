6.아래의 에러가 나는 이유를 설명하고 수정 하시오.

int a = 3;
int b = 4;
	
double result = a / b;

System.out.println(result);

0.0 으로 출력
int 연산결과는 int로 출력되며 소수점 아래가 소멸되는데, 그 값을 result로 대입하면서 double로 자동 형변환되어 .0이 붙게된다.
원하는 결과값인 0.75를 얻기 위해서는 최소한 a,b 둘 중 하나는 double로 선언되거나, 연산 전에 명시적 형변환이 되어야 한다.
cf)정수형 상수가 연산에 포함되는 경우 3.0 등의 소수점을 포함하여 연산에 사용하면 자동으로 double로 인식한다.

8. 아래의 출력 결과를 확인하고, 130이 나오는 이유를 설명
System.out.println('A'); 
System.out.println('A' + 'A'); 

A / 130
연산 시에 char형이 아닌 int형으로 우선 간주하기 때문에 계산된 결과를 아스키코드 값으로 변환해서 계산
따라서 대문자 A의 아스키코드값 65를 2번 더한 결과인 130이 출력

(추가)
'A'(리터럴)를 뿌리기 위해 메모리에 방을 잡는데, 이 때 데이터타입 char, 메모리 값으로 아스키코드 값인 65를 사용한다.
'A'+'A'의 결과값을 받기 위해서는 일단 'A'를 메모리에 올리는 인코딩 과정이 필요하다.
이 때 코드값인 65가 메모리에 올라가고, CPU연산 시 int형으로 자동 형변환이 이루어져서 65+65=130 의 결과가 출력된다.
